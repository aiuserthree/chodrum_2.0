
# 드럼 악보 다운로드 사이트 상세 설계서 (모바일 최적화)

> FO(사용자 화면) / BO(관리자 화면) 분리 기준으로 작성합니다.

# 1. 문서 개요 및 공통 정의

## 1.1 페이지 ID 체계
- FO 페이지: `FO-XX-YY` (예: FO-05-01 = 장바구니 > 목록)
- BO 페이지: `BO-XX-YY` (예: BO-02-01 = 악보관리 > 등록)
- 팝업/모달: 각 페이지 ID 뒤 `-Pnn` (예: FO-03-01-P01)

## 1.2 사용자 상태 정의
| 코드 | 상태 | 설명 |
|------|------|------|
| GUEST | 비로그인 | 로그인하지 않은 방문자 |
| MEMBER_EMAIL | 일반 회원 | 이메일 인증 가입 (비밀번호 보유) |
| MEMBER_SOCIAL | 소셜 회원 | 구글/네이버/카카오 연동 (비밀번호 없음) |
| NONMEMBER_BUYER | 비회원 구매자 | 비회원으로 결제 완료, 이메일로 조회 |
| ADMIN | 관리자 | BO 접근 권한자 |

## 1.3 주요 데이터 상태값 정의
| 구분 | 상태값 |
|------|--------|
| 주문(Order) | PENDING(결제대기) / PAID(결제완료) / CANCELED(취소) / REFUNDED(환불) |
| 결제(Payment) | READY / PAID / FAILED / CANCELED / PARTIAL_REFUND / REFUND |
| 다운로드 권한(Entitlement) | ACTIVE(기간내) / EXPIRED(기간만료) / REVOKED(회수) |
| 악보(Sheet) | ON_SALE(판매중) / STOP(판매중지) / HIDDEN(숨김) |
| 회원(Member) | ACTIVE(정상) / SUSPENDED(정지) / WITHDRAWN(탈퇴) |

## 1.4 핵심 공통 정책 (전 페이지 적용)
- 모든 결제는 **반드시 장바구니를 경유**한다. (직접 결제 경로 없음)
- 다운로드 가능 기간은 **결제 완료 시점 기준 7일(168시간)**.
- 다운로드 파일 형식은 **PDF**.
- 비회원의 유일한 조회 키는 **주문 시 입력한 이메일**.
- 소셜 회원은 비밀번호를 보유하지 않으며, 관련 메뉴는 미노출.

# 2. 공통 컴포넌트 / 공통 정책

## 2.1 공통 UI 컴포넌트
| 컴포넌트 | 위치 | 설명 |
|----------|------|------|
| 상단 검색바 | 전 페이지 상단 고정 | 악보명/아티스트 검색, 자동완성 |
| 하단 네비게이션 바 | 전 페이지 하단 고정 | 홈 / 검색 / 장바구니 / 마이페이지 |
| 장바구니 뱃지 | 하단 네비 장바구니 아이콘 | 담긴 수량 표시 |
| Sticky 결제 버튼 | 상세/장바구니/결제 하단 고정 | 주요 CTA 노출 |
| 다운로드 잔여기간 뱃지 | 구매내역/조회 화면 | 색상 단계 표시(7~4일 녹색 / 3~2일 주황 / 1일 이하 빨강) |
| 토스트/스낵바 | 전역 | 담기/삭제/오류 등 즉시 피드백 |

## 2.2 공통 정책 컴포넌트
- **로그인 세션 유지**: 회원은 토큰 기반 세션 유지.
- **비회원 장바구니/찜**: 브라우저 세션(로컬) 기반 (정책 확정 필요, 6장 참조).
- **약관 동의**: 결제/가입 시 필수 동의 항목과 선택 동의 항목 분리.
- **에러 처리 원칙**: 사용자 귀책(입력 오류)과 시스템 귀책(PG 오류) 메시지 분리.

# 3. FO 페이지 상세 설계

---

## FO-01. 홈 (메인)

**페이지 목적**: 서비스 진입점, 검색/추천/카테고리 탐색 시작점.

**화면 구성 요소**
| 영역 | 요소 | 설명 |
|------|------|------|
| 상단 | 검색바 | 곡명/아티스트 입력, 자동완성 |
| 배너 | 추천 배너 | BO에서 설정한 프로모션/추천 악보 노출 |
| 리스트1 | 추천 악보 | 관리자 큐레이션 |
| 리스트2 | 신규 악보 | 최근 등록순 |
| 리스트3 | 인기 악보 | 판매량/조회수 기준 |
| 하단 | 카테고리 바로가기 | 장르/난이도별 진입 |
| 하단 고정 | 네비게이션 바 | 홈/검색/장바구니/마이 |

**인터랙션**
- 검색어 입력 → 자동완성 노출 → 선택 시 FO-02(목록)로 이동.
- 악보 카드 탭 → FO-03(상세)로 이동.
- 찜 버튼 탭 → 즉시 찜 반영(토스트).

**상태별 노출**
- GUEST/회원 공통 노출. 로그인 여부에 따라 하단 '마이페이지' 진입 동작만 분기.

---

## FO-02. 악보 목록

**페이지 목적**: 필터/정렬/검색으로 원하는 악보 탐색.

**화면 구성 요소**
| 영역 | 요소 | 설명 |
|------|------|------|
| 필터 | 카테고리/난이도/장르 | 다중 선택 가능 |
| 정렬 | 인기순/최신순/가격순/이름순 | 단일 선택 |
| 리스트 | 악보 카드 | 썸네일, 곡명, 아티스트, 가격, 찜 버튼 |
| 페이징 | 무한스크롤 or 더보기 | 모바일 최적화(무한스크롤 권장) |

**인터랙션**
- 필터/정렬 변경 → 리스트 즉시 갱신.
- 카드 탭 → FO-03(상세).
- 찜 버튼 탭 → 찜 토글.

**예외 처리**
- 검색 결과 없음 → "검색 결과가 없습니다" 안내 + 추천 악보 노출.

---

## FO-03. 악보 상세페이지

**페이지 목적**: 악보 정보 확인 및 장바구니 담기.

**화면 구성 요소**
| 영역 | 요소 | 설명 |
|------|------|------|
| 미리보기 | 워터마크 미리보기 | 일부 페이지만 노출(범위 정책 필요) |
| 정보 | 곡명/아티스트/난이도/장르/페이지수 | 기본 메타 정보 |
| 가격 | 판매 가격 | 할인 시 원가/할인가 |
| 수량 | 수량 선택 UI | 터치 친화적 +/- 버튼 |
| CTA | 찜하기 / 장바구니 담기 | 하단 Sticky 배치 |
| 추천 | 관련 악보 | 동일 장르/난이도 추천 |

**인터랙션 / 비즈니스 규칙**
- **직접 결제 버튼 없음** (반드시 장바구니 경유).
- '장바구니 담기' → 장바구니 반영 + 뱃지 갱신 + 토스트("장바구니에 담았습니다").
- 이미 장바구니에 있는 악보 재담기 시 → 수량 합산 또는 안내(정책 확정 필요).
- 판매중지/숨김 상태 악보는 상세 진입 불가 또는 "판매 종료" 안내.

**예외 처리**
- 존재하지 않는 악보 ID 접근 → 404 안내 페이지.

---

## FO-04. 찜 목록 (위시리스트)

**페이지 목적**: 관심 악보 저장 및 장바구니 이동.

**화면 구성 요소**
- 찜한 악보 리스트(썸네일/곡명/가격)
- 개별/전체 '장바구니 담기'
- 개별/전체 '찜 해제'

**비즈니스 규칙**
- 회원: 계정에 저장(로그인 시 동기화).
- 비회원: 세션 기반 저장(정책 확정 필요).
- 장바구니 담기 시 찜 목록 유지(자동 삭제하지 않음, 정책 권장).

---

## FO-05. 장바구니

**페이지 목적**: 결제의 필수 관문. 최종 구매 목록 확정.

**화면 구성 요소**
| 영역 | 요소 | 설명 |
|------|------|------|
| 리스트 | 담긴 악보 | 썸네일/곡명/수량/가격 |
| 수량 | 수량 수정 | +/- 조절, 삭제 |
| 선택 | 선택/전체 체크박스 | 선택 결제 지원 |
| 금액 | 총 결제 금액 | 선택 상품 기준 실시간 계산 |
| CTA | 결제하기 | 하단 Sticky |

**비즈니스 규칙**
- 선택 결제 및 전체 결제 지원.
- 결제하기 클릭 시 로그인 상태 분기:
  - 회원 → FO-06 주문/결제 (정보 자동 입력)
  - GUEST → 로그인/비회원 구매 선택 유도 → FO-06

**예외 처리**
- 장바구니 비어있음 → 결제 버튼 비활성화 + "담긴 악보가 없습니다".
- 담긴 악보가 판매중지됨 → 해당 항목 결제 제외 + 안내.

---

## FO-06. 주문 / 결제

**페이지 목적**: 주문자 정보 확인 및 PG 결제 진행.

**화면 구성 요소**
| 영역 | 회원 | 비회원 |
|------|------|--------|
| 주문자 정보 | 자동 입력 | 이름/연락처 입력 |
| 이메일 | 계정 이메일 | **다운로드 조회용 이메일 필수 입력** |
| 주문 상품 | 목록/금액 확인 | 동일 |
| 결제 수단 | PG 연동 선택 | 동일 |
| 약관 동의 | 필수 동의 | 동일 |
| CTA | 결제하기 | 결제하기 |

**비즈니스 규칙**
- **장바구니 미경유 직접 진입 차단** (URL 직접 접근 시 장바구니로 리다이렉트).
- 비회원 이메일은 오입력 시 조회 불가 → 입력 확인 UX(재확인/확인 문구) 강조.
- 결제 완료 → FO-07 이동, **결제 완료 시점부터 7일 카운트 시작**.

**예외 처리**
- 결제 실패 → 장바구니 유지, 재결제 유도.
- 결제 중 이탈 → 주문 PENDING 처리, 일정 시간 후 자동 만료.

---

## FO-07. 결제 완료

**페이지 목적**: 결제 결과 확인 및 다운로드 경로 안내.

**화면 구성 요소**
- 주문번호 / 결제 내역(상품/금액/일시)
- 회원: "마이페이지 > 구매내역/다운로드" 이동 버튼
- 비회원: "비회원 주문 조회" 안내 + 입력 이메일 재확인 안내

**비즈니스 규칙**
- 다운로드 권한(Entitlement) ACTIVE 생성, 만료 시각 = 결제완료 + 168h.
- 회원은 즉시 마이페이지에 반영.

---

## FO-08. 회원가입 / 로그인

### FO-08-01. 로그인
- 이메일 + 비밀번호 로그인
- 소셜 로그인(구글/네이버/카카오)
- '비밀번호 찾기'(일반 회원 전용) 링크

### FO-08-02. 회원가입 (일반)
1. 이메일 입력 → 인증(코드/링크) 발송
2. 인증 완료 시 해당 이메일 = 로그인 ID 확정
3. 비밀번호 설정(필수)
4. 약관 동의 → 가입 완료

### FO-08-03. 회원가입 (소셜)
1. 소셜 버튼 클릭 → 플랫폼 인증
2. 반환된 이메일/계정 = 로그인 ID 확정
3. 비밀번호 없음
4. 약관 동의 → 가입 완료

**비즈니스 규칙 / 예외 처리**
- 미인증 이메일은 가입 불가.
- 동일 이메일이 일반/소셜 중복 시 → 기존 가입 방식 안내(정책 확정 필요).
- 소셜 로그인 실패/취소 → 로그인 화면 복귀 + 안내.

---

## FO-09. 마이페이지 (회원 전용)

**접근 권한**: MEMBER_EMAIL / MEMBER_SOCIAL (GUEST 접근 시 로그인 유도)

### FO-09-01. 구매 내역 / 다운로드
| 요소 | 설명 |
|------|------|
| 구매 악보 리스트 | 결제 완료 악보 |
| 다운로드 버튼 | 기간 내 ACTIVE 시 활성 |
| 잔여 기간 표시 | "남은 기간: N일" 뱃지 |
| 만료 처리 | 7일 경과 시 버튼 비활성 + 안내 문구 |

**규칙**: 기간 내 재다운로드 허용(권장, 정책 확정 필요).

### FO-09-02. 찜 목록
- FO-04와 동일 데이터, 마이페이지 내 진입 경로.

### FO-09-03. 내 정보 수정
- 일반 회원: 기본 정보 + 비밀번호 변경 가능.
- 소셜 회원: 기본 정보만, **비밀번호 변경 메뉴 미노출**.

### FO-09-04. 회원 탈퇴
- 탈퇴 시 잔여 다운로드 권한 처리 정책 안내(정책 확정 필요).
- 탈퇴 확인 절차(비밀번호 재확인 / 소셜 재인증).

---

## FO-10. 비회원 주문 조회 (비회원 전용)

**페이지 목적**: 비회원 구매자의 구매 악보 조회 및 다운로드.

**화면 흐름**
1. 주문 시 입력한 이메일 입력
2. 이메일 인증(인증코드/인증링크 — 정책 확정 필요)
3. 인증 완료 → 해당 이메일의 구매 악보 리스트 노출
4. 다운로드 버튼(기간 내 활성) + 잔여 기간 표시

**예외 처리**
- 이메일 오입력/구매 이력 없음 → "해당 이메일로 주문 내역이 없습니다".
- 인증 실패 → 재발송 유도.
- 기간 만료 → 다운로드 버튼 비활성 + 안내.

# 4. BO 페이지 상세 설계

---

## BO-01. 대시보드
| 위젯 | 설명 |
|------|------|
| 매출 현황 | 일/주/월 매출, 결제 건수 |

# 5. 비즈니스 로직 시나리오

> 형식: [시나리오명] → 선행조건 → 단계별 흐름 → 결과 → 예외

## 시나리오 S-01. 일반 회원가입 후 첫 구매·다운로드

**선행조건**: 비로그인 신규 방문자(GUEST)

**흐름**
1. 홈 진입 → 검색바에 곡명 입력 → 자동완성에서 선택.
2. 악보 목록에서 원하는 악보 카드 탭 → 상세페이지 진입.
3. 수량 선택 후 '장바구니 담기' → 장바구니 뱃지 +1, 토스트 노출.
4. 하단 네비 '장바구니' 진입 → 결제 대상 선택 → '결제하기'.
5. 비로그인 상태이므로 로그인/회원가입/비회원 구매 선택 노출.
6. '회원가입' 선택 → 이메일 입력 → 인증코드 발송 → 인증 완료.
7. 비밀번호 설정 → 약관 동의 → 가입 완료(자동 로그인).
8. 주문/결제 페이지에서 주문자 정보 자동 입력 → 결제 수단 선택 → 약관 동의 → 결제.
9. 결제 완료 → 다운로드 권한(ACTIVE, 만료=+168h) 생성.
10. 마이페이지 > 구매내역에서 다운로드 버튼 활성 + "남은 기간: 7일" 표시.
11. 다운로드 버튼 탭 → PDF 저장.

**결과**: 회원 계정에 구매 이력·다운로드 권한 저장.

**예외**
- 6단계 이메일 미인증 → 가입 중단, 가입 불가.
- 8단계 결제 실패 → 장바구니 유지, 재결제 유도.

---

## 시나리오 S-02. 비회원 구매 및 이메일 조회 다운로드

**선행조건**: 로그인/가입 의사 없는 GUEST

**흐름**
1. 상세페이지 → '장바구니 담기' → 장바구니 진입 → '결제하기'.
2. 로그인 유도 화면에서 '비회원 구매' 선택.
3. 주문/결제 페이지에서 주문자 정보 + **다운로드 조회용 이메일 필수 입력**.
4. 이메일 재확인 UX(오입력 방지) → 결제 수단 선택 → 약관 동의 → 결제.
5. 결제 완료 → 해당 이메일 기준 다운로드 권한(ACTIVE) 생성.
6. 결제 완료 화면에서 "비회원 주문 조회" 안내.
7. 이후 'FO-10 비회원 주문 조회' 진입 → 이메일 입력 → 인증(코드/링크).
8. 인증 완료 → 구매 악보 리스트 + 다운로드 버튼(잔여 기간) 노출.
9. 다운로드 → PDF 저장.

**결과**: 이메일이 유일한 조회 키로 사용되어 다운로드 가능.

**예외**
- 3단계 이메일 오입력 → 이후 조회 불가(경고 UX로 사전 방지).
- 7단계 인증 실패 → 재발송 유도.

---

## 시나리오 S-03. 소셜 간편가입 및 구매

**선행조건**: GUEST, 소셜 계정 보유

**흐름**
1. 장바구니 → 결제하기 → 로그인 화면.
2. 소셜 로그인(구글/네이버/카카오) 선택 → 플랫폼 인증.
3. 최초 이용 시 소셜 간편가입 처리(이메일/계정=로그인 ID, 비밀번호 없음) → 약관 동의.
4. 주문/결제 페이지 자동 진입 → 정보 자동 입력 → 결제.
5. 결제 완료 → 다운로드 권한 생성 → 마이페이지 반영.

**결과**: 소셜 회원으로 가입·구매 완료.

**예외**
- 마이페이지 > 내 정보 수정 진입 시 **비밀번호 변경 메뉴 미노출**.
- 동일 이메일 일반 가입 이력 존재 시 → 기존 가입 방식 안내(정책 확정 필요).

---

## 시나리오 S-04. 다운로드 기간(7일) 만료

**선행조건**: 결제 완료 후 168시간 경과

**흐름**
1. 사용자가 마이페이지(또는 비회원 조회) 진입.
2. 시스템이 현재시각과 만료시각(결제완료+168h) 비교.
3. 만료 시 Entitlement EXPIRED 처리 → 다운로드 버튼 비활성화.
4. "다운로드 가능 기간이 만료되었습니다" 안내 문구 노출.

**결과**: 다운로드 불가 상태.

**예외 / 후속**
- 사용자 문의 → 관리자 BO-05에서 재부여 가능(사유 기록).
- 재부여 시 Entitlement ACTIVE 전환 + 새 만료시각 설정.

---

## 시나리오 S-05. 결제 실패 및 재결제

**선행조건**: 장바구니에 상품 존재, 결제 시도

**흐름**
1. 주문/결제 페이지에서 결제 진행.
2. PG 오류/한도 초과/사용자 취소 등으로 결제 실패.
3. 주문 상태 PENDING/FAILED 처리, **장바구니 상품은 유지**.
4. "결제에 실패했습니다. 다시 시도해주세요" 안내.
5. 재결제 유도 → 성공 시 정상 완료 흐름 진입.

**결과**: 장바구니 데이터 손실 없이 재시도 가능.

---

## 시나리오 S-06. 환불 및 다운로드 권한 회수 (BO)

**선행조건**: 결제 완료 주문 존재, 환불 요청 접수

**흐름**
1. 관리자가 BO-03 주문/결제 관리에서 대상 주문 조회.
2. 환불 처리(전체/부분) 실행.
3. 결제 상태 REFUNDED 전환.
4. 정책에 따라 해당 악보 Entitlement REVOKED → FO 다운로드 버튼 비활성.
5. 사용자에게 환불/다운로드 회수 안내(이메일 알림).

**결과**: 환불 완료 및 다운로드 권한 회수(정책 연동).

**예외**
- 이미 다운로드 완료된 건의 회수 범위는 정책 확정 필요(6장 참조).

---

## 시나리오 S-07. 장바구니 미경유 결제 차단

**선행조건**: 사용자가 주문/결제 URL 직접 접근 시도

**흐름**
1. FO-06 주문/결제 페이지 직접 접근 감지.
2. 유효한 장바구니 세션/선택 상품 없음 확인.
3. 장바구니(FO-05)로 리다이렉트 + "장바구니를 통해 결제를 진행해주세요" 안내.

**결과**: 모든 결제는 장바구니 경유 원칙 강제.

---

## 시나리오 S-08. 악보 판매중지 후 기구매자 처리

**선행조건**: 특정 악보 STOP/HIDDEN 전환

**흐름**
1. 관리자가 BO-02에서 해당 악보 상태 변경.
2. FO 목록/검색/상세 노출 제외.
3. 이미 구매한 사용자의 Entitlement는 **유지**(기간 내 다운로드 가능).
4. 장바구니에 담겨있던 미결제 사용자는 결제 시 해당 항목 제외 + 안내.

**결과**: 신규 판매 중단, 기구매자 권리 보호.
