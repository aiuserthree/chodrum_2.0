
# 드럼 악보 다운로드 사이트 설계서 초안 (모바일 최적화)

## 1. 서비스 개요

### 1.1 서비스 정의
- 드럼 악보를 검색·선택하여 온라인 결제 후 PDF 형태로 다운로드할 수 있는 모바일 최적화 웹 서비스
- 일반 쇼핑몰과 유사한 구매 흐름(찜 → 장바구니 → 결제 → 다운로드) 제공
- 회원/비회원 모두 구매 가능

### 1.2 핵심 특징
- 모바일 우선(Mobile First) 반응형 설계
- 회원/비회원 구매 지원
- 소셜 간편가입(구글, 네이버, 카카오) 및 일반 이메일 가입 지원
- 결제 완료 후 7일간 한정 다운로드 정책
- 사용자 화면(FO)과 관리자 화면(BO) 분리 운영

## 2. 사용자 유형 정의

| 구분 | 설명 | 로그인 ID | 비밀번호 | 마이페이지 |
|------|------|-----------|----------|-----------|
| 일반 회원 | 이메일 인증 가입 | 인증한 이메일 주소 | 필수 설정 | 있음 |
| 구글 간편회원 | 구글 계정 연동 | 구글 이메일/계정 | 없음 | 있음 (비밀번호 수정 불가) |
| 네이버 간편회원 | 네이버 계정 연동 | 네이버 이메일/계정 | 없음 | 있음 (비밀번호 수정 불가) |
| 카카오 간편회원 | 카카오 계정 연동 | 카카오 이메일/계정 | 없음 | 있음 (비밀번호 수정 불가) |
| 비회원 | 가입 없이 구매 | 없음 (구매 시 이메일 입력) | 없음 | 없음 (이메일 인증으로 조회) |

## 3. 정보 구조 (IA) 및 메뉴 구조

### 3.1 사용자 화면 (FO) 메뉴 구조

```
[FO] 드럼 악보 다운로드 사이트
│
├── 1. 홈 (메인)
│   ├── 상단 검색바 (악보 검색)
│   ├── 추천/신규/인기 악보 리스트
│   └── 카테고리 바로가기
│
├── 2. 악보 목록
│   ├── 카테고리/난이도/장르 필터
│   ├── 정렬 (인기순/최신순/가격순/이름순)
│   ├── 검색 결과 리스트
│   └── 악보 카드 (썸네일, 곡명, 아티스트, 가격, 찜 버튼)
│
├── 3. 악보 상세페이지
│   ├── 악보 미리보기 (일부 페이지 워터마크 미리보기)
│   ├── 곡 정보 (곡명, 아티스트, 난이도, 장르, 페이지 수)
│   ├── 가격 정보
│   ├── 수량 선택
│   ├── 찜하기 버튼
│   ├── 장바구니 담기 버튼
│   └── 관련 악보 추천
│
├── 4. 찜 목록 (위시리스트)
│   ├── 찜한 악보 리스트
│   ├── 장바구니 담기
│   └── 찜 해제
│
├── 5. 장바구니
│   ├── 담긴 악보 리스트 (수량 수정/삭제)
│   ├── 선택/전체 결제
│   ├── 총 결제 금액
│   └── 결제하기 버튼 → 주문/결제 페이지
│
├── 6. 주문/결제
│   ├── [회원] 주문자 정보 (자동 입력)
│   ├── [비회원] 주문자 정보 + 이메일 입력 (다운로드 조회용)
│   ├── 주문 상품 확인
│   ├── 결제 수단 선택 (PG 연동)
│   ├── 약관 동의
│   └── 결제 진행 → 결제 완료 페이지
│
├── 7. 결제 완료
│   ├── 주문 번호 / 결제 내역
│   ├── [회원] 마이페이지 이동 안내
│   └── [비회원] 이메일 인증 조회 안내
│
├── 8. 회원가입 / 로그인
│   ├── 로그인
│   │   ├── 이메일 + 비밀번호 로그인
│   │   └── 소셜 로그인 (구글/네이버/카카오)
│   ├── 회원가입
│   │   ├── 일반 가입 (이메일 인증 + 비밀번호 설정)
│   │   └── 소셜 간편가입 (구글/네이버/카카오)
│   └── 비밀번호 찾기 (일반 회원만)
│
├── 9. 마이페이지 (회원 전용)
│   ├── 9-1. 구매 내역 / 다운로드
│   │   ├── 결제 완료 악보 리스트
│   │   ├── 다운로드 버튼 (결제 후 7일 이내 활성화)
│   │   ├── 다운로드 가능 잔여 기간 표시
│   │   └── 7일 경과 시 다운로드 버튼 비활성화
│   ├── 9-2. 찜 목록
│   ├── 9-3. 내 정보 수정
│   │   ├── 기본 정보 수정
│   │   └── 비밀번호 변경 (일반 회원만, 소셜 회원 비노출)
│   └── 9-4. 회원 탈퇴
│
└── 10. 비회원 주문 조회 (비회원 전용)
    ├── 주문 시 입력한 이메일 입력
    ├── 이메일 인증 (인증코드/인증링크)
    ├── 구매 악보 리스트 확인
    └── 다운로드 버튼 (결제 후 7일 이내 활성화)
```

## 4. 핵심 비즈니스 로직

### 4.1 회원가입 / 로그인

**일반 회원가입**
1. 이메일 주소 입력 → 이메일 인증(인증코드/인증링크 발송)
2. 인증 완료 시 해당 이메일이 로그인 ID로 확정
3. 비밀번호 설정(필수)
4. 약관 동의 후 가입 완료

**소셜 간편가입 (구글/네이버/카카오)**
1. 소셜 로그인 버튼 클릭 → 해당 플랫폼 인증
2. 플랫폼에서 받은 이메일/계정이 로그인 ID로 확정
3. 비밀번호 없음
4. 약관 동의 후 가입 완료

**로그인 규칙**
- 일반 회원: 이메일 + 비밀번호
- 소셜 회원: 각 플랫폼 소셜 로그인
- 동일 이메일이 일반/소셜에 중복될 경우 처리 정책 필요 (예: 기존 가입 방식 안내)

### 4.2 구매 흐름 (공통 규칙)

**핵심 규칙: 모든 결제는 반드시 장바구니를 거쳐야 함**
1. 악보 검색 또는 목록에서 선택
2. 상세페이지 진입 → 수량 선택
3. 찜하기(선택) 또는 장바구니 담기
4. 장바구니에서 결제 진행 (직접 결제 경로 없음)
5. 주문/결제 페이지에서 PG사 연동 결제
6. 결제 완료 → 다운로드 권한 부여

**찜 목록**
- 결제 전 관심 악보 저장
- 찜 목록에서 장바구니로 이동 가능
- 회원: 계정에 저장 / 비회원: 세션 기반(정책 결정 필요)

**장바구니**
- 결제의 필수 관문
- 수량 수정, 개별/전체 삭제 가능
- 선택 결제 및 전체 결제 지원

### 4.3 결제

- 회원: 주문자 정보 자동 입력, 결제 완료 후 마이페이지에 즉시 반영
- 비회원: 주문 시 이메일 입력 필수 (추후 다운로드 조회의 유일한 수단)
- PG사 연동을 통한 온라인 결제
- 결제 완료 시점부터 7일 다운로드 기간 카운트 시작

### 4.4 다운로드 정책 (핵심)

- 결제 완료된 악보에 한해 다운로드 버튼 생성
- **다운로드 가능 기간: 결제 완료 후 7일(168시간)**
- 다운로드 버튼 옆 잔여 기간 표시 (예: "남은 기간: 3일")
- 7일 경과 시 다운로드 버튼 자동 비활성화 → 다운로드 불가
- 다운로드 형식: PDF (사용자 로컬 PC에 저장)
- 기간 내 재다운로드 허용 여부는 정책 결정 필요 (권장: 기간 내 재다운로드 허용)
- 관리자는 예외적으로 다운로드 기간 재부여 가능(BO)

### 4.5 구매 악보 조회

**회원**
- 마이페이지 > 구매 내역/다운로드에서 결제 완료 악보 리스트 확인
- 각 악보별 다운로드 버튼 및 잔여 기간 표시

**비회원**
- 별도 "비회원 주문 조회" 메뉴 진입
- 주문 시 입력한 이메일 입력 → 이메일 인증
- 인증 완료 시 해당 이메일의 구매 악보 리스트 확인 및 다운로드

### 4.6 내 정보 수정
- 일반 회원: 기본 정보 + 비밀번호 변경 가능
- 소셜 회원: 기본 정보만 가능, 비밀번호 변경 메뉴 미노출(비활성)

## 5. 주요 화면별 상태/예외 처리

| 상황 | 처리 방식 |
|------|-----------|
| 다운로드 기간(7일) 경과 | 다운로드 버튼 비활성화 + 안내 문구 |
| 결제 실패 | 장바구니 유지, 재결제 유도 |
| 비회원 이메일 오입력 | 주문 조회 불가 안내 (이메일 정확성 강조) |
| 소셜 회원 비밀번호 변경 시도 | 메뉴 미노출로 원천 차단 |
| 미인증 이메일 가입 시도 | 인증 완료 전 가입 불가 |
| 장바구니 미경유 결제 시도 | 결제 진입 차단(장바구니 유도) |

---

## 6. 정책 결정 필요 항목 (초안 검토 포인트)

1. 비회원 찜 목록 지원 여부 (세션 기반 vs 미지원)
2. 다운로드 기간 내 재다운로드 허용 및 횟수 제한 여부
3. 동일 이메일의 일반 가입 / 소셜 가입 중복 처리 정책
4. 환불 시 다운로드 권한 회수 정책
5. 미리보기 제공 범위 (몇 페이지, 워터마크 적용 여부)
6. 비회원 이메일 인증 방식 (인증코드 vs 인증링크)
7. 다운로드 파일 보안 (워터마크/DRM 적용 여부)

## 7. 모바일 최적화 고려사항

- 하단 고정 네비게이션 바 (홈/검색/장바구니/마이페이지)
- 검색바 상단 고정 및 자동완성
- 악보 카드형 리스트 (썸네일 중심)
- 장바구니/결제 버튼 하단 고정 (Sticky)
- 터치 친화적 수량 조절 UI
- 다운로드 잔여 기간 시각적 강조(뱃지/컬러)
- 소셜 로그인 버튼 상단 배치로 간편가입 유도
